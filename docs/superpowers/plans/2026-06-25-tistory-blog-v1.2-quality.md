# tistory-blog 스킬 v1.2 품질 개선 (구현 완료 기록)

> 이 문서는 **이미 적용된** 변경의 기록이다. 코드 자체는 `~/.hermes/skills/tistory-blog/`(git 밖 런타임 디렉토리)에 있어 PR diff로 담기지 않으므로, 변경 결정·근거·검증을 이 문서로 추적한다. (v1.1 기록: `2026-06-11-blog-quality-enhancement.md`)

**Goal:** 수동으로 작성한 고품질 기술 글(nginx/Gateway API 정리글) 수준으로 자동 생성 초안 품질을 끌어올린다.

**Architecture:** 전부 `~/.hermes/skills/tistory-blog/` 내부(지침 md 3 + 헬퍼 py 2 + test 2). 이대리(NestJS) 변경 0 — 릴레이 계약(`NOTION_URL:` 출력, `BLOG_NOTIFY_SLACK` 가드) 불변. stdlib only, stdlib `unittest`.

---

## 문제 (Why)

자동 생성된 nginx 초안(`Notion '블로그 초안' DB`)을 사용자가 수동으로 전면 재작성하며 6대 약점을 확인했다. 각 약점의 근본 원인을 스킬에서 특정:

| # | 증상 | 근본 원인 (파일:위치) |
|---|---|---|
| 1 | 회고체라 공허 (안 겪은 주제를 1인칭 경험담으로) | `voice.md` "선배 경험담·1인칭 나·~했다"를 **전 카테고리 강제** |
| 2 | 내용 얕음 (회피성 단언, 코드 1블록, 최신사실 누락) | `SKILL.md §2` 스니펫 리서치 + `§3.5` 루브릭 자기채점이 느슨 |
| 3 | 기계적 4단 구조(개념/동작/예시/정리) | `templates.md` 기술·CS 골격 고정 |
| 4 | 제목후보·`루브릭 88/100` 메타가 본문 상단 노출 | `SKILL.md §4`(검증메타 prepend) + `create_notion_draft.py`(제목후보 callout) |
| 5 | 콜아웃·표·토글 없이 평문+코드만 | `create_notion_draft.py md_to_blocks` 가 heading/bullet/code/quote만 지원 |
| 6 | 용어 설명 없어 불친절 | voice/templates에 풀어쓰기 지침 없음 |

추가로 `lint_voice.py` 분량 상한이 일괄 1500자(코드 포함 측정)라, 깊은 기술 글이 자동으로 "분량 위반" 처리되는 문제도 확인.

## 목표 (What)

- CS·기술 글은 경험 날조 없이 객관 정리체 + 풀어쓰기로 깊이를 확보한다.
- 본문에 메타(제목후보·검증점수)를 노출하지 않는다.
- Notion 블록(콜아웃/표/토글/구분선)으로 가독성을 살린다.
- 위 변경이 기존 결정론 게이트(lint/verify/test)를 깨지 않는다.

## 변경 (How)

### 1단계 — 지침(md)
- [x] **`voice.md`**: 카테고리별 톤 분기 추가 — 회고·코테는 1인칭 경험담, **CS·기술은 객관 정리체(겪지 않은 경험 날조 금지)**. 회피성 단언 금지, 용어 첫 등장 시 풀어쓰기, "왜"까지 적기.
- [x] **`templates.md`**: 기술·CS 4단 고정 → "훅 도입 + 유연 구조". 공통 뼈대(훅/깊이/가독성/정직) 섹션 신설.
- [x] **`SKILL.md`**: §2 웹 리서치에 "최신 상태 핵심사실(버전/EOL/GA) 확인" 요구, §3 카테고리 톤 선택, §3.5 루브릭 강화(회피단언 감점·최신사실 반영), **§4 검증메타·제목후보 본문 노출 제거** → §6 터미널 보고 / 페이지 맨 아래 토글로.

### 2단계 — 코드(py) + 테스트
- [x] **`create_notion_draft.py`**: `md_to_blocks` 확장 — 콜아웃(`:::callout <emoji> <color>`), 토글(`:::toggle 제목`), 표(`| a | b |`), 인라인 `**굵게**`·`` `코드` `` 지원. 제목후보 callout(본문 상단) → `📝 다른 제목 후보` 토글(페이지 맨 아래)로 이동.
- [x] **`lint_voice.py`**: 분량 상한 카테고리별(CS·Nest 6000 / 회고 4000 / 코테 3000 / 기본 1500) + **코드블록·`:::` 마커 제외**(산문 기준 측정, 콜아웃 아이콘 이모지 오탐 방지). CLI `--category` 추가.
- [x] **`test_create_notion_draft.py` / `test_lint_voice.py`**: 제목후보 토글·콜아웃·표·토글·인라인·카테고리 상한·코드제외 테스트 추가.

## 동작 변화 / 리스크

- 본문 마크다운 문법이 확장됨(`:::callout`, `:::toggle`, 표). 기존 문법(heading/bullet/code 등)은 그대로 동작 — 하위호환.
- `lint_voice`의 분량 기준이 "코드 포함 1500" → "산문 기준 카테고리별"로 바뀜. 코드 없는 글은 사실상 동일, 코드 많은 기술 글만 통과폭이 넓어짐.
- 이대리 릴레이 계약·발행 정책 변화 **없음**.

## 범위 밖 / 후속
